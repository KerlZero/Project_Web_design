# Requirements

Requirements are the source of truth between backlog ideas and delivery work.

## Current Mode

The project is currently in `v1 bugfix` mode.

- Bugfix work may proceed against the current v1 design and implementation.
- Future feature backlog must remain on hold until the project owner explicitly starts feature work.
- SA/BA Agent is responsible for deciding whether an incoming request is a bug, feature, design clarification, or asset request.

## Requirement Package

Each requirement version should contain:

- Requirement brief
- User stories
- Functional requirements
- Non-functional requirements
- Acceptance criteria
- Risks, assumptions, and open questions

Recommended structure:

```text
Requirements/
  v1/
    current-bugfix-scope.md
  feature-name-v2/
    requirement-brief.md
    user-stories.md
    acceptance-criteria.md
```
