# Future Feature Backlog

Ideas to keep for later versions. Do not implement these in v1.

## 1. Add Component By AI Agent

### 1.1 Solution: Sync Component Add To Web

- Let an AI agent generate a new component record.
- Sync generated component data into the website component library.
- Support fields such as name, category, tags, HTML, CSS, JavaScript, notes, and preview.
- Add review step before saving to the component data source.

### 1.2 Add From Clip Or URL

- User sends a video clip or URL.
- AI agent analyzes the source.
- AI agent extracts possible UI components, layout ideas, or interaction patterns.
- System converts the result into component records.
- User reviews before saving.

## 2. Login Website

- Add authentication before editing component records.
- Keep browsing open for guest users if needed.
- Add account-level saved snippets later.

## 3. Support Full Website Design

- Expand from individual components into full website patterns.
- Store complete page layouts, sections, navigation structures, and design systems.
- Add full-page preview examples.

## 4. Delete Component

- Allow removing component records.
- Add confirmation modal.
- Consider archive/restore before permanent delete.

## 5. Website Permissions

Possible roles:

- Admin
- Guest
- Dev
- UX/UI Design

Permission ideas:

- Admin can add, edit, delete, and manage data.
- Guest can view and copy examples.
- Dev can add/edit code-focused components.
- UX/UI Design can add/edit design notes, visual references, and layout guidance.
